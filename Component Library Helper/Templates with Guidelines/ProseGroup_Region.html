ProseGroup Region holds and lays out a collection of other components in a special grid. 

Individual components provide content and styling. The ProseGroup Region assembles them and handles responsive layout for different user devices, such as tablets or phones. 

It is not possible to use these components outside the region. 

Layout

On large viewports, ProseGroup Region has three columns. (Details in requirements doc.) Each column holds specific components. These columns stack on small screens.

Column interactions

The columns can negatively interact in ways an author can prevent. 

Avoid using full width (FW) components at the top of the main column IF using the left or right columns. Certain main column components spread out of the column to fill the entire page, meaning they overlap the other two columns. Since the left and right columns are usually quite short, this overlap is intentional and provides both aesthetic and practical benefits.

Full width components are:

  • Prose block quote
  • Prose full-width illustration 

Template limitations

Not all templates support all components. The list above is everything available anywhere. 

Several of the components available in ProseGroup Region are unique to certain areas of the site, and simply won't show up elsewhere. For example, the Prose author bio is unique to Blogs and the Success story metadata is unique to Customer Success Stories. 

Visual Style

Guidelines

  • Break up long sections of text with interesting content. (Exception: legal pages, like the privacy policy.) Add an image with Prose full-width illustration or Prose inline media player, or highlight key points with Prose block quote. 

Placement

  • Use in the main content region.
  • Can be used multiple times on a page, although this is exceptional.
