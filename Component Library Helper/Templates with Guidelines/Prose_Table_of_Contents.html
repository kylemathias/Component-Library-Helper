An index of the sections in a ProseGroup Region. 

Table of Contents is part of a collection of components which must be used inside a ProseGroup Region, which provides layout via a special grid.

Use when

  • A ProseGroup Region's content is long enough to warrant a table of contents for the user to skip to a particular section. 
  • Encouraging social sharing. 
  • Offering an alternative format of an article, particularly a PDF. 
  • Offering a call to action intrinsic to an article's subject, such as an event description with the CTA "Register now". 

Don't use when

  • Creating a page without a ProseGroup Region. TOC is for articles only. 
Basic:
All optional features active:

Editorial

Heading (optional, unique, default value)

  • Plain text field
  • Default value in English: "Table of contents" (translate this default). This can be overridden by the editor. 

Anchor to Major Heading (optional, repeats, default automated)

  • By default, nav items are populated by parsing the page for major headings.
  • Each item can have an editorial alias. An editor could shorten the heading "Five warning signs that you have a public cloud spending problem" to the nav item label "Five warning signs". This is optional. 

Anchor to Minor Heading (optional, repeats, default automated)

  • Identical to Anchor to Major Heading, except it is for minor headings. 
  • Shown indented under the corresponding major heading in the component. 
  • Setting: show minor headings on/off (if on display parse and display sub-items | if off hide/ignore sub-items)
  • When the TOC is set to be sticky, sub-items are visually collapsed until the corresponding major heading's section is in view, when they reveal. 

RTE Block (optional)

  • Rich text field (use the simplest configuration: bold and bullets encouraged)
  • If desired, the TOC can be blank and this block can be used as a content sidebar. This is an edge-case without strict design requirements. Currently used this way in the press release template. 

CTA Button (optional)

  • 0-1 buttons. Default is no button. 
  • Character limit is 25. Shorter is better. 
  • Primary scenario: download the PDF equivalent of the current page. 

(Not available yet) Social Share (optional)

  • A template sets this element on or off for all its pages. Initially, this is a blogs only feature. 

Behavior

Sticky

The author (or a template) can toggle Table of Contents to stay fixed while the article scrolls, called "sticky". 

  • In normal mode (default), the component is a standard block element that scrolls with the page (i.e. is NOT sticky). 
  • In sticky mode, the component stays fixed to the top of the viewport, while the content scrolls. Example: LUCI's right-side page nav.

Sticky mode can conflict with certain other components. Always visually test your page when using TOC sticky mode to ensure there are no conflicts. 

Sticky TOC safe components:

  • Any component with "Prose" in its name, e.g. Prose Block Quote or Prose Inline Media Player. 
  • Exception: Be careful with the Prose Full-Width Illustration. Keep the image's point of interest centered or to the right, never on the left where TOC may cover it. 

Sticky TOC conflicts:
If you use these components within the ProseGroup Region, you may not use TOC in sticky mode. 

  • Card Band
  • Card Band Images
  • Side x Side
  • Feature Tiles

Responsiveness

  • The index of sections is hidden on small screens. 
  • The RTE block displays at all sizes. 
  • The CTA displays at all sizes. 
  • The social links display at all sizes.
