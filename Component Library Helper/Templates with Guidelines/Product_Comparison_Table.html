Display product details side by side for easy comparison.

 

Template Block

 

You can copy and paste the table into your template directly from netapp.com page, it should format correctly and you can then have a SME edit the table within the NewsCred content editor

 

 

Use When

  • Helping to compare features for a range of similar products.

Do not Use When

  • Presenting a general overview of all products. Consider the Showcase Product. 
  • Navigation to several products. Consider Image with Tiles or Tabbed Band Tiles. 
  • Displaying a simple table. Consider RTE - Table. 

 

 

Editorial

The component contains a single table arranged with products as columns and attributes as rows. The table has footnotes after it. 

 

Table title (required, hidden)

  • Name what is being compared, usually the models from a product line. For example, "Compare SolidFire Cloud Storage Model & Specs".
  • Character limit is 45.

 

Table head, thead

Product image row (required)

    • Always the first row.
    • Contains one product photo for each column.

Product name row (required)

    • Each cell after the first contains a product name, linked to that product's page (content and link: both required)
    • Short name of product. Remove the series name if needed for space. For example, in a comparison of SolidFire models, the NetApp SolidFire H610S-1 should be truncated to "H610S-1".
    • Character limit is 13. Avoid wrapping.
    • Order products from largest/most capable to smallest/least capable. On smaller screens, the right-most columns will require scrolling. 

Table body, tbody

Product attribute row (required)

    • Use a product attribute row to differentiate products (columns) by a single attribute. 
    • row header (required)  
      • Label every row. 
      • Character limit is 36.
      • RTE field. Footnotes okay. Avoid other formatting. Styling will be overridden/ignored.
    • row cells (required)
      • Fill every cell. Use "n/a" instead of leaving a cell blank. 
      • Character limit is 216. 
      • RTE field. Paragraphs, links, and bullets okay. Footnotes okay. Avoid other formatting. 
    • Span columns for shared attributes. Do not span into the heading column; use it to label the shared attribute.
    • Prefer footnotes in the row heading to redundant footnotes in individual cells. 

Section row (optional)

    • Use a section row to group related product attribute rows in long comparison tables. 
    • Character limit is 13. 
    • If using more than 20 product attribute rows, consider providing a "Key Attributes" section. 

Sub-dividing columns (optional, unusual)

    • Subdivide product columns when presenting different workloads or configurations, such as "Typical" versus "Maximum". Consider whether to add full columns to the table instead. Are the new columns products? 
    • Be cognizant of horizontal space. Subdividing more than three products creates a very wide table that will require the user to horizontally scroll on most devices. 

Footnotes

  • Use footnotes sparingly. 
  • Footnotes may appear in product attribute rows, either in the row header cell (preferred) or in individual attribute cells. 
  • Footnotes may contain links.
